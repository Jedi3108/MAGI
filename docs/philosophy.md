# MAGI Philosophy

MAGI is built around one question:

How should an AI system reason before it acts?

MAGI rejects the idea that one model should silently produce one final answer. Instead, it exposes disagreement, uncertainty, trade-offs, and reasoning paths.

The system is inspired by the idea of multiple enduring perspectives forming a council, but its purpose is practical: transparent decision support for local AI assistants.

## Core Belief

Important decisions benefit from structured disagreement.

## Design Values

- Transparency
- Local-first operation
- Model independence
- Reproducible decisions
- Separation of reasoning and execution
- Respect for uncertainty
