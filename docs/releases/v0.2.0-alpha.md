# MAGI v0.2.0-alpha

Release type: alpha  
Status: local-first deliberation engine with real Ollama execution and structured Chair reliability.

## Summary

MAGI v0.2.0-alpha moves the project beyond the initial protocol skeleton.

This release validates that MAGI can run a complete local deliberation cycle using a real Ollama model, produce structured council rounds, recover from imperfect model JSON, show protocol progress, and generate a more reliable non-voting Chair dossier using an authoritative final vote record.

## Major changes since v0.1.0-alpha

### Real local model readiness

- Added Ollama readiness checks.
- Added installed model listing.
- Added local model resolution.
- Added friendlier errors for missing or unavailable Ollama models.
- Verified real execution with `llama3.2`.

### JSON-mode robustness

- Added Ollama JSON-mode support.
- Added robust JSON extraction helpers.
- Added parser fallbacks for malformed JSON-like model output.
- Prevented broken raw JSON from leaking into terminal fields.
- Added tests for malformed model responses.

### Protocol progress indicators

- Added optional progress callbacks to `MagiEngine`.
- Added CLI progress messages for each protocol stage.
- Real model runs no longer appear frozen while the protocol is executing.

### Council prompt quality

- Strengthened permanent council personas.
- Improved Round 1 independent analysis prompts.
- Improved cross-examination prompts.
- Improved satisfaction evaluation prompts.
- Improved reflection consistency constraints.
- Improved Chair dossier instructions.
- Added self-question prevention.
- Added prompt quality tests.

### Structured Chair final vote record

- Added an authoritative final reflected vote record for the Chair.
- Included final vote counts and per-member final votes.
- Included final confidence, original vote, learned statement, and final reason.
- Updated Chair instructions to prioritize final reflected votes over transcript inference.
- Improved no-consensus summaries and final vote attribution.

## Current capabilities

MAGI can now:

- Run a full local deliberation protocol.
- Use deterministic mock mode for testing.
- Use a real local Ollama model for council reasoning.
- Execute:
  - Round 1: independent analysis
  - Round 2: cross-examination
  - Round 3: satisfaction evaluation
  - Round 4: reflection
  - Final decision tally
  - Non-voting Chair dossier
- Preserve structured final vote information for the Chair.
- Report protocol progress during slow model runs.
- Maintain passing unit tests across the protocol.

## Known limitations

- Reasoning quality is still limited by the local model used.
- All council members may still use the same model unless multiple models are installed.
- The Chair still summarizes prose, though it now receives authoritative final vote data.
- MAGI has no long-term memory yet.
- MAGI has no external tools or worker execution layer yet.
- No UI exists yet.
- This remains an alpha research system, not a production decision-maker.
