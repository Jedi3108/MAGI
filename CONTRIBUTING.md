# Contributing to MAGI

MAGI is developed as a serious local deliberative intelligence project.

## Branch Workflow

- `main` is stable.
- `develop` is active development.
- `feature/*` branches are used for focused changes.

## Commit Guidelines

Each commit should answer:

1. What capability did this add?
2. Why was this design chosen?
3. How do we know it works?

Prefer meaningful commit messages.

Example:

```text
Document initial deliberation protocol
Project Principle

Do not add features only because they are interesting.

A feature belongs in core MAGI only if it improves decision quality, transparency, reliability, or extensibility.

Experimental ideas belong in experiments/.
